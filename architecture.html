Global Filter System

4 filters are managed by the useFilters composable (singleton). All views subscribe to the same reactive state.

Time Period
API param: month · Format: YYYY-MM or Q1-2025
Orders Dashboard not Inventory
Warehouse / Location
API param: warehouse · San Francisco, London, Tokyo
Inventory Orders Dashboard
Category
API param: category · Circuit Boards, Sensors, Actuators…
Inventory Orders Dashboard
Order Status
API param: status · Delivered, Shipped, Processing, Backordered
Orders Dashboard not Inventory

Views (Pages)

Dashboard KPI cards, revenue charts, cost breakdown — all filters
Inventory Stock levels, reorder points, search — warehouse + category only
Orders Order list by status, expandable items — all 4 filters
Spending Revenue vs cost charts, expense categories, transactions
Demand Forecast table with trend indicators — no filters
Backlog Shortage management with priority levels — no filters
Reports Quarterly performance, monthly revenue trends

Composables (Shared Logic)

useFilters.js

Singleton reactive state for all 4 global filters. Provides getCurrentFilters() to build API params.

selectedPeriod, selectedLocation, selectedCategory, selectedStatus hasActiveFilters (computed) · resetFilters() · getCurrentFilters()
useAuth.js

Mock authentication. Provides current user, initials, and task list. Bilingual user data.

currentUser (computed) · getInitials() · logout()
useI18n.js

Translation system for EN/JA. Persists locale to localStorage. Switches currency (USD/JPY).

t(key, params) · currentLocale · currentCurrency · setLocale()
